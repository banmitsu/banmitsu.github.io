VMware Workstation 12 Pro Performance increment

  • Anonymous
  • 2016/07/07
  • Comments
  • How-To, VMware

As I try to train my model using keras, backend=theano on my VM, OS: ubuntu 14.04, I found the performance is dreadful by default setting. Therefore, I found some useful instructions to speed up VM.

Disk I/O / vRAM

The host’s memory is, e.g, 32GB ram.
Assign 10GB to vm (1/3)

(1) Edit vm.vmx

  • MemTrimRate=0, disable memory trimming

  • Workstation uses a memory trimming technique to de-allocate unused virtual machine memory for the host to re-allocate. Trimming usually has little impact on performance, and it may be needed in low memory situations. However, memory trimming can interfere with disk-oriented performance in a guest.

  • 設定MemTrimRate=0後，原本分配給vm的記憶體閒置後將不會被host回收，所以vm會運作得較快速。
    (p.s. 若mainMem.useNamedFile=”TRUE”,memory trimming可能會對影響disk I/O)

  • sched.mem.pshare.enable = “FALSE”, disable page sharing

  • VMware uses a page sharing technique to allow guest memory pages with identical contents to be stored as a single copy-on-write page. Page sharing decreases host memory usage, but consumes system resources, potentially including I/O bandwidth.

  • 停用memory sharing後，vm就停止比對記憶體，將相同的內容存在同一個page內來達到減用vm memory的使用量。

  • mainMem.useNamedFile = “FALSE”

  • MemAllowAutoScaleDown = “FALSE”

(2) Edit C:\ProgramData\VMware\VMware Workstation\config.ini

  • prefvmx.minVmMemPct = “100”

  • prefvmx.useRecommendedLockedMemSize = “TRUE”

CPU

Inspect host’s CPU, (e.g., i7-6700, 8 cores)
number of processors: 1
number of cores per processor: 4-6
preferred mode: Inter VT-x/EPT or AMD-V/RVI

  • Virtualize Inter VT-x/EPT or AMD-V/RVI checked.
  • Virtualize CPU performance counters checked.
